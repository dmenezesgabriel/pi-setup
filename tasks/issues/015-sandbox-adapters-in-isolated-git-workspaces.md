# Task: Sandbox all benchmark adapters in isolated per-trial git workspaces

## Priority

P0 — Skill-generated artifacts are actively polluting the real repository directory. Every benchmark trial run risks creating untracked `issues/` and `docs/adrs/` files in the repo, corrupting the real planning history and making behave evaluations unreliable (agent writes to the real repo, snapshot is empty, behave sees nothing).

## Dependencies

- No task dependency; this is a self-contained bug fix inside `benchmarks/harness/adapters/`.
- No ADR dependency; the isolation mechanism (temp dir + `git init`) is an additive change with no cross-cutting architecture impact.

## Assignability

**AFK** — root cause is fully diagnosed, fix is straightforward: create an isolated git workspace per trial and point every adapter's subprocess CWD at it.

## Context

Coding agents (pi, claude, opencode) anchor their working directory to the nearest git repository root. When a benchmark trial runs:

- `OpenCodeAdapter` hardcodes `cwd=_REPO_ROOT` — every file the agent writes goes directly into the pi-setup repository.
- `ClaudeCodeAdapter` passes no `cwd` at all — the agent inherits whoever's working directory launched the benchmark (typically also the repo root or benchmarks dir).
- `PiAgentAdapter` correctly creates a temp dir and passes `cwd=workspace`, but does not run `git init` in it. Because `--skill <path>` points to a path inside the real repo, pi may walk the directory tree upward, detect the real git root, and write files there instead of the temp CWD.

Observed symptoms: after a benchmark run, `docs/adrs/` and `issues/` under the pi-setup repo root contain files whose names match benchmark task topics (rate limiting, multi-tenancy, Redis queues) — content generated by the agent, not by the human planning workflow. These files are untracked but present on disk, polluting future `git status`, confusing the plan-it skill's `issues-lock.json` counter, and making behave assertions incorrect (the agent wrote files outside the snapshot workspace, so `workspace_snapshot` is empty and all file-existence scenarios fail).

The fix is a shared `_make_workspace()` helper that:
1. Creates a temporary directory.
2. Runs `git init` in it (giving the agent a git root to anchor to, preventing escape up the real tree).
3. Pre-creates the output directories the skill expects (`issues/`, `docs/adrs/`, `implementation/`).
4. Returns the `Path`.

Every adapter uses this helper and passes `cwd=str(workspace)` to its subprocess. After the run, every adapter calls the existing `_snapshot_workspace()` logic (currently only in `PiAgentAdapter`) to capture what the agent wrote.

Because `ClaudeCodeAdapter` and `OpenCodeAdapter` currently do not capture workspace files — they only parse stdout — they also need workspace snapshot collection added so their `TrialResult.workspace_snapshot` is populated for behave evaluation.

## Use Cases

- **Feature**: Isolated benchmark trial execution
- **Scenario**: Benchmark run produces no new untracked files in the real repository
- **Given** a benchmark trial runs `plan-it` via any adapter
- **When** the trial completes
- **Then** no new files exist under `<repo-root>/issues/` or `<repo-root>/docs/adrs/`
- **And** `TrialResult.workspace_snapshot` contains the files the agent wrote in the isolated workspace

## Definition of Ready

- `benchmarks/harness/adapters/pi_agent.py`, `claude_code.py`, and `opencode.py` are read and understood.
- The shared snapshot helper `_snapshot_workspace()` in `pi_agent.py` is known to work correctly.
- `git` is available in PATH on all machines that run benchmarks.

## Functional Requirements

- `FR-001`: Extract a `_make_workspace() -> Path` helper into `benchmarks/harness/adapters/_workspace.py`. It creates a temp directory, runs `git init -q` in it, and pre-creates `issues/`, `docs/adrs/`, and `implementation/` subdirectories.
- `FR-002`: Extract `_snapshot_workspace(workspace: Path) -> dict[str, str]` into the same `_workspace.py` module so all three adapters share one implementation.
- `FR-003`: `PiAgentAdapter.run()` calls `_make_workspace()` (replacing the current inline temp dir + mkdir logic) and retains `cwd=str(workspace)`.
- `FR-004`: `ClaudeCodeAdapter.run()` calls `_make_workspace()`, passes `cwd=str(workspace)` to `subprocess.run`, and calls `_snapshot_workspace(workspace)` after the subprocess exits to populate `TrialResult.workspace_snapshot`.
- `FR-005`: `OpenCodeAdapter.run()` calls `_make_workspace()`, replaces `cwd=str(_REPO_ROOT)` with `cwd=str(workspace)`, and calls `_snapshot_workspace(workspace)` after the subprocess exits.
- `FR-006`: All three adapters use a `with tempfile.TemporaryDirectory() as tmp` context so the isolated workspace is cleaned up automatically after each trial, regardless of success or failure.
- `FR-007`: The `git init -q` in `_make_workspace()` uses `--initial-branch=main` (or equivalent) to suppress the "hint: default branch name" warning that would pollute subprocess stderr.

## Non-Functional Requirements

- `NFR-001`: `_make_workspace()` must complete in under 200 ms (git init on a fresh empty directory is fast; this must not become a bottleneck across hundreds of trials).
- `NFR-002`: No adapter may reference `_REPO_ROOT` as the subprocess CWD after this fix.

## Observability Requirements

- `OBS-001`: After any benchmark trial, `git status` in the real repo must show no new untracked files under `issues/`, `docs/adrs/`, or `implementation/`.
- `OBS-002`: `TrialResult.workspace_snapshot` must be non-empty for any trial where the agent successfully wrote files (verifiable in MLflow artifact `raw_output.txt` or behave pass rate > 0).

## Acceptance Criteria

- `AC-001`: **Given** a benchmark trial runs via `PiAgentAdapter`, **When** the trial completes, **Then** no new files appear in the real repo directory.
- `AC-002`: **Given** a benchmark trial runs via `ClaudeCodeAdapter`, **When** the trial completes, **Then** no new files appear in the real repo directory and `TrialResult.workspace_snapshot` is populated.
- `AC-003`: **Given** a benchmark trial runs via `OpenCodeAdapter`, **When** the trial completes, **Then** no new files appear in the real repo directory and `TrialResult.workspace_snapshot` is populated.
- `AC-004`: **Given** the trial workspace is created, **When** `ls -la <workspace>` is run, **Then** a `.git/` directory exists.
- `AC-005`: **Given** the trial workspace contains a `.git/` directory, **When** the agent runs `git rev-parse --show-toplevel`, **Then** it returns the temp workspace path (not the real repo root).
- `AC-006`: **Given** the temp workspace context exits (normally or via exception), **When** the cleanup runs, **Then** the temp directory and all agent-generated files inside it are deleted.

- `UT-002`: Inside the workspace, run `git rev-parse --show-toplevel` and assert the output equals the workspace path (not the real repo root). Covers `FR-007`, `AC-005`.

- `OpenCodeAdapter` no longer references `_REPO_ROOT` as CWD.

- Regression test `REG-001` passes.
- After a real benchmark run, `git status` on the real repo shows no untracked files under `issues/`, `docs/adrs/`, or `implementation/`.
